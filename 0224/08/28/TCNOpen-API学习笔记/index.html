TCNOpen API学习笔记

0224-08-28 2024-09-05

TRDP 是一种基于 TCP/IP 的通信协议，用于在列车控制系统中进行数据交换。它提供了一种可靠、高效的方式，使列车控制系统能够与其他设备进行通信，实现列车的自动化控制和数据采集。
在TRDP（TraCommunication Network Open Protocol）中，“tlc” 和 “tlp” 通常指的是协议栈中的两个不同层次。

tlc：代表 “TraCommunication Link Control”，即列车通信链路控制层。这一层负责处理数据链路层的功能，包括数据帧的封装、传输错误的检测和纠正、流量控制等。在TRDP中，tlc层提供了可靠的数据传输服务。

tlp：代表 “TraCommunication Protocol”，即列车通信协议层。这一层负责处理网络层的功能，包括路由选择、分组转发、网络地址转换等。在TRDP中，tlp层提供了列车网络中的通信协议。

TLc_init 和 tlp_openSession 这两个调用实际上代表了在TRDP协议栈中不同层次的初始化步骤。tlc_init 函数用于初始化tlc层的资源，而 tlp_openSession 函数用于打开一个tlp层的会话，这可能涉及到网络地址的配置、连接的建立等操作。

以下是一些关键点的总结：

tlc层负责数据链路层的功能。
tlp层负责网络层的功能。
tlc_init 用于初始化tlc层。
tlp_openSession 用于打开tlp层的会话
TRDP 最小 DEMO：

tlc_init 函数：

  • 作用：初始化TRDP协议栈的底层（tlc），包括内存配置和调试打印函数。
  • 参数：
    • pPrintDebugString：指向调试打印函数的指针。
    • pRefCon：用户提供的上下文指针，用于调试打印函数。
    • pMemConfig：指向内存配置结构体的指针。

tlc_openSession 函数：

  • 作用：打开一个与TRDP堆的会话，为应用程序分配一个句柄。
  • 参数：
    • pAppHandle：指向应用程序会话句柄的指针。
    • ownIpAddr：本地设备的IP地址。
    • leaderIpAddr：领导者设备的IP地址。
    • pMarshall：指向编组配置结构体的指针。
    • pPdDefault：指向默认过程数据（PD）配置结构体的指针。
    • pMdDefault：指向默认消息数据（MD）配置结构体的指针。
    • pProcessConfig：指向进程配置结构体的指针。
  • 作用：用于在 TRDP 网络中发送请求消息。
  • 参数：
    • appHandle: 应用程序会话的句柄。
    • subHandle: 订阅的句柄。
    • serviceId: 服务 ID，标识请求的服务类型。
    • comId: 通信 ID，标识请求的通信内容。
    • etbTopoCnt: ETB（Ethernet Train Backbone）拓扑计数，指定消息在 ETB 网络中的传播路径。
    • opTrnTopoCnt: 操作传输拓扑计数，指定消息在操作传输网络中的传播路径。
    • srcIpAddr: 源 IP 地址，即发送请求的设备的 IP 地址。
    • destIpAddr: 目标 IP 地址，即请求消息要发送到的设备的 IP 地址。
    • redId: 冗余 ID，用于在冗余网络中标识冗余路径。
    • pktFlags: 数据包标志，用于指定数据包的特殊属性，如优先级、确认要求等。
    • pData: 指向数据的指针，请求消息中携带的数据。
    • dataSize: 数据的大小，以字节为单位。
    • replyComId: 回复通信 ID，指定期望的回复消息的通信内容。
    • replyIpAddr: 回复 IP 地址，指定期望的回复消息的发送地址。

tlp_subscribe 函数：

  • 作用：订阅过程数据（PD）包。
  • 参数：
    • appHandle：应用程序会话句柄。
    • pSubHandle：指向订阅句柄的指针。
    • pUserRef：用户提供的参考值，将与信息结构一起返回。
    • pfCbFunction：指向订阅者特定回调函数的指针。
    • serviceId：可选的服务ID，默认值为0。
    • comId：要接收的数据包的COM ID。
    • etbTopoCnt：ETB拓扑计数，用于本地通信时为0。
    • opTrnTopoCnt：操作拓扑计数，用于方向敏感通信时不为0。
    • srcIpAddr1：源IP地址，地址范围的较低地址。
    • srcIpAddr2：源IP地址，地址范围的较高地址。
    • destIpAddr：要加入的IP地址。
    • pktFlags：数据包标志，如默认、无、编组、回调等。
    • timeout：超时时间，以微秒为单位。
    • toBehavior：超时行为。

tlp_publish 函数：

  • 作用：发布过程数据（PD）包。
  • 参数：
    • appHandle：应用程序会话句柄。
    • pPubHandle：指向发布句柄的指针。
    • pUserRef：用户提供的参考值，将与信息结构一起返回。
    • pfCbFunction：指向发布者特定回调函数的指针。
    • serviceId：可选的服务ID，默认值为0。
    • comId：要发送的数据包的COM ID。
    • etbTopoCnt：ETB拓扑计数，用于本地通信时为0。
    • opTrnTopoCnt：操作拓扑计数，用于方向敏感通信时不为0。
    • srcIpAddr：源IP地址。
    • destIpAddr：目标IP地址。
    • interval：发布间隔时间。
    • redId：冗余ID。
    • pktFlags：数据包标志，如默认、无、编组、回调等。
    • pData：指向要发送的数据的指针。
    • dataSize：数据大小，以字节为单位。

tlp_getInterval 函数：

  • 作用：获取过程数据（PD）包的最低发送间隔。
  • 参数：
    • appHandle：应用程序会话句柄。
    • pInterval：指向时间间隔结构体的指针。
    • pFileDesc：指向文件描述符集合的指针。
    • pNoDesc：指向套接字数目的指针。

vos_select 函数：

  • 作用：设置提供的套接字集合中的就绪套接字。
  • 参数：
    • highDesc：最高描述符值加一。
    • pReadableFD：指向可读文件描述符集合的指针。
    • pWriteableFD：指向可写文件描述符集合的指针。
    • pErrorFD：指向错误文件描述符集合的指针。
    • pTimeOut：指向超时时间结构体的指针。

tlc_process 函数：

  • 作用：TRDP处理器的工作循环，处理发送和接收队列中的PD和MD包。
  • 参数：
    • appHandle：应用程序会话句柄。
    • pRfds：指向文件描述符集合的指针。
    • pCount：指向处理计数的指针。

tlp_put 函数：

  • 作用：更新要发送的过程数据（PD）。
  • 参数：
    • appHandle：应用程序会话句柄。
    • pubHandle：发布句柄。
    • pData：指向要发送的数据的指针。
    • dataSize：数据大小，以字节为单位。

tlp_get 函数：

  • 作用：从订阅句柄中获取过程数据（PD）。
  • 参数：
    • appHandle：应用程序会话句柄。
    • subHandle：订阅句柄。
    • pPdInfo：指向过程数据（PD）信息结构的指针。
    • pData：指向接收数据的指针。
    • pDataSize：指向接收数据大小的指针。

vos_threadDelay 函数

  • 作用：延迟当前线程的执行，延迟时间为给定的微秒数。
  • 参数：
    • delay：延迟时间，以微秒为单位。
  • 在TRDP（Train Communication Network Open Protocol）中，MD（Message Data）和PD（Process Data）：
  • Message Data：
    • 消息数据是指在TRDP网络中传输的非周期数据；
    • 消息数据通常用于传输控制信息、数据、命令等；
    • MD的发送和接收不基于固定的时间间隔，而是根据需求；
    • 当列车系统中某个设备需要发送一个状态更新，或接收一个配置命令时，就会使用MD进行通信。
  • Process Data：
    • 过程数据是指在TRDP网络中传输的周期数据；
    • 用于传输实时性要求较高的过程控制数据，如传感器数据、控制信号等；
    • PD消息的发送和接收是基于固定的时间间隔，通常是毫秒级的；
    • 列车上的速度传感器会定期发送PD消息，以确保控制系统能够实时获取列车的速度信息。
